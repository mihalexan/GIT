<1> Как скопировать на свой компьютер репозиторий из github.com, что
    должно быть сделано предварительно?

Предварительно на GitHub должен быть добавлен SSH key.
1. На GitHub: Cоздать или выбрать нужный репозиторий, перейти в него,
   нажать на кнопку "Code", выбрать SSH, скопировать ссылку.
2. Создать папку на ноутбуке
3. На GitBash: перейти в выбранную папку используя команду 'cd (путь/к/файлу)'. 
4. Выполнить команду клонирования 'git clone' и вставить 
   скопированную ссылку репозитория SSH после этой команды.
5. Нажать Enter, дождаться пока команда git clone скопирует
   репозиторий с GitHub на компьютер.

<2> Как получить на локальный компьютер изменения с github.com?

1. В Git Bash перейти в папку, где находится локальный репозиторий 
   используя команду 'cd (путь/к/файлу)'. 
2. Убедиться, что вы находитесь на нужной ветке можно используя 
   команду git branch для просмотра списка веток и команду git checkout 
   для переключения на нужную ветку, если это необходимо.
3. Выполнить команду git pull, чтобы извлечь изменения с GitHub и 
   объединить их с вашей локальной веткой.
4. Если нужно получить изменения из другой ветки, укажите ее имя после команды. 
5. Git выполнит операцию извлечения (fetch) и объединения (merge)
   изменений из удаленного репозитория в вашей локальной ветке. Если 
   нет конфликтов слияния, изменения будут автоматически добавлены 
   к вашим локальным файлам.
6. После выполнения этих шагов ваш локальный репозиторий будет обновлен 
   с изменениями с GitHub. Локальные файлы будут синхронизированы с 
   последним состоянием удаленного репозитория.

<3> Какую информацию нам дают команды : 'git status', git log', 'git remote -v'.

- команда 'git status' дает информацию о состоянии репозитория Git. 
  Показывает, какие файлы были изменены, добавлены или удалены в
  локальном репозитории с момента последнего коммита.
  Сообщает о текущей ветке, на которой вы находитесь, и о наличии
  нерезолвленных конфликтов слияния.
- команда 'git log' позволяет просмотреть историю коммитов в вашем 
  репозитории Git. Она выводит список коммитов, начиная с последнего и 
  до самого первого коммита. Для каждого коммита выводится информация, 
  такая как хэш коммита, автор, дата и время коммита, а также 
  сообщение коммита.
- команда 'git remote -v' проверяет связан ли репозиторий с GitHub.
  Если команда выводит 2 строки с (fetch) получать с GitHub (push)
  отправлять на GitHub это говорит о том что репозиторий привязан к 
  GitHub или к другому сервере который находится в этом сообщении. 
  После этого можно убедиться что репозиторий был получен с GitHub  
  (в нашем случае).
